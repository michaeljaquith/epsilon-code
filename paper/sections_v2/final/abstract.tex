Large language models generate code that looks uniformly confident. A correct
call to \texttt{stripe.PaymentIntent.create()} and a deprecated call to
\texttt{stripe.Charge.create()} are emitted with the same surface tone, even
when the underlying token distribution is a near-50/50 split (probability
$0.52$ vs.\ $0.47$). Code-generation uncertainty is primarily \emph{intra-generational}: visible
in the token distribution at generation time, invisible once tokens are
committed. Multi-sample methods, which detect inter-generational
inconsistency, therefore produce almost no signal on code tasks.

We introduce \eps, a per-token normalized Shannon entropy computed from the
top-$k$ log-probabilities returned by the generation API, aggregated over the
semantic tokens of a function via AST filtering and max-pooling. \eps is
designed as a \emph{recall-maximizing filter}, not a detector. On a
30-prompt calibration benchmark across GPT-4o, GPT-4o-mini, and GPT-4-turbo,
\eps at the \textsc{flagged} threshold ($\eps \ge 0.30$) achieves recall
$=1.00$ on true failures with precision $=0.272$. The black-box multi-sample
baseline UQLM achieves $5\%$ HIGH detection at $0\%$ LOW FP, versus \eps's $85\%$ HIGH detection at $50\%$ LOW FP on GPT-4o;
a \texttt{min\_logprob} baseline achieves $45$--$55\%$ where \eps achieves
$70$--$85\%$.

The low stage-1 precision is the design. We pair the filter with a parallel
LLM review loop that reclassifies each flagged entry as \textsc{cleared} or
\textsc{confirmed}. On 201 production and scenario entries across three
models, the two-stage system lifts reviewer precision above $94\%$ (Wilson
lower bound; $n=57$, zero late false positives in the 2B configuration). A
context-accumulated variant, which injects model-specific learned false-%
positive patterns into the reviewer, clears $68.7\%$ of flagged entries
with zero late false positives. The contribution is the system: a filter
that misses nothing, paired with a reviewer that keeps up with machine-%
speed code generation.
