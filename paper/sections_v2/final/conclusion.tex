\section{Conclusion}
\label{sec:conclusion}

We introduced \eps, a per-token normalized Shannon entropy over top-$k$
log-probabilities, AST-filtered and max-aggregated into a single score
per generated function. \eps is a filter, not a detector: it achieves the recall target
established in \S\ref{subsec:pr}. The contribution is the two-stage system.
Pairing a recall-maximizing filter with a parallel LLM reviewer
produces an architecture in which nothing uncertain ships unreviewed
and the reviewer scales with generation speed.

The framing matters. Multi-sample uncertainty methods produce almost
no signal on code generation because code uncertainty is intra-%
generational --- visible in the single-generation token distribution,
invisible across re-samples. Once the distinction is named, the
design of \eps follows directly. The same mechanism --- early-token
commitment forcing later tokens --- plausibly applies in other
structured-output domains (SQL, configuration files, typed program
synthesis); empirical verification is left to future work. Generation
is fast; review is the bottleneck. A filter that misses nothing plus
a parallel reviewer moves review off the developer's critical path.
